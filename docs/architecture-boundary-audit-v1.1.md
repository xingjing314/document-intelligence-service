# DIS v1.1 架构最终审计（Architecture Boundary Audit）

**版本**：v1.1 最终边界审计版  
**审计类型**：Permanent Boundary Freeze Audit  
**审计日期**：2025  
**审计焦点**：单一问题 —— StructuredDocument 是否已真正成为 DIS 的规范产物（Canonical Document Model）

**审计原则**：
- 仅讨论系统边界与职责
- 严禁讨论实现、技术、代码、目录、API 或工程方案
- 目标：冻结 DIS 的永久边界（Permanent Boundary）

---

## 审计前提（已冻结边界）

DIS = Document Recovery Service

核心职责链：**Recover → Normalize → Assemble → Trace**

唯一输出：**StructuredDocument**

DIS 严禁进入 Knowledge Engineering 领域。

---

# 问题1 审计：StructuredDocument 是否仍存在“中间产物思维”？

## 审计结论

**存在残留的中间产物思维风险，但程度已大幅降低。**

在 v0.1.0 及早期冻结设计中，StructuredDocument（当时名为 StructuredKnowledge）被明显设计为“知识工程的输入准备阶段”。即使在边界修正后，部分描述仍隐含“它是为下游 Chunk/Concept 准备的中间表示”。

**当前状态诊断**：

| 方面                     | 残留中间产物思维表现                              | 严重程度 | 说明 |
|--------------------------|--------------------------------------------------|----------|------|
| 命名与定位               | 仍有人习惯性地称其为“知识抽取的前置步骤”          | 中       | 名称已修正，但认知惯性仍存 |
| 完备性目标               | 部分讨论仍聚焦“是否足够支持 RAG”，而非“是否完整恢复教材事实” | 中       | 目标锚点存在偏移 |
| Provenance 的角色        | 有时被视为“方便下游做知识引用”，而非“文档恢复的内在要求” | 低-中    | 动机偏差 |
| 资产描述的深度           | 偶尔出现“生成适合教学理解的描述”倾向              | 低       | 已在边界修正中收敛 |

**核心判断**：
StructuredDocument **尚未完全** 确立为“教材事实的最终规范化表示（Canonical Representation）”。它仍带有“为知识工程做准备”的潜意识定位，而非“教材被最大程度忠实恢复后的唯一权威形态”。

# 问题2：Minimum Canonical Document Model (MCDM)

## 审计结论

StructuredDocument 必须被定义为 **Minimum Canonical Document Model (MCDM)** —— 即一个教材经过 DIS 处理后，所能达到的**最小但完备的规范形态**。

下游任何组件（TeachingChunker、KnowledgeStructurer、Embedding Service、Retriever、Agent）都**不应**需要回退到原始 PDF 进行二次解析。所有后续工作都必须以 StructuredDocument 为唯一事实来源。

## MCDM 一级对象职责定义

StructuredDocument 作为 MCDM，必须包含以下**一级对象**（按职责而非实现方式划分）：

### 1. Document Identity & Provenance
- 教材的唯一身份标识
- 原始来源元数据（文件名、页数、处理时间、处理版本等）
- 整个文档的恢复处理溯源记录

### 2. Structural Hierarchy
- 教材的完整逻辑章节树（Chapter / Section / Subsection）
- 每个节点的标题、层级、页码范围
- 目录恢复结果与正文实际结构的对应关系

### 3. Logical Content Flow
- 按阅读顺序组织的逻辑内容序列
- 内容块的类型分类（正文、标题、列表、引用等）
- 跨页、跨章节的连续性保持

### 4. Asset Collection
- 所有被恢复的视觉与结构化资产（Figure、Table、Formula、Diagram 等）
- 每个资产的独立身份与基础恢复信息
- 资产与正文内容的引用关系（双向）

### 5. Cross Reference Network
- 教材内部所有显式和可推断的交叉引用关系
- “见图 X-X”“如表 Y-Y 所示”“公式 Z-Z”等引用链路
- 引用关系的强度与类型分类

### 6. Element-Level Provenance
- 每一个可独立引用的对象（结构节点、内容块、资产、引用）都必须携带完整的来源追溯信息
- 包括物理位置（页码、坐标）、恢复来源、置信度、恢复过程记录

### 7. Document Integrity Manifest
- 文档整体恢复质量的结构化声明（哪些部分恢复完整、哪些存在降级、整体置信度分布）
- 缺失或无法恢复内容的显式标注

**MCDM 的本质要求**：
以上七类对象共同构成一个“自洽的、封闭的、可独立使用”的文档规范模型。任何下游系统拿到 StructuredDocument 后，都能完整重建教材的结构、内容组织、视觉资产位置及所有引用关系，而无需接触原始 PDF。

# 问题3 审计：Asset Recovery Layer 的越界风险

## 审计结论

Asset Recovery Layer 是当前设计中最容易产生**隐性知识工程越界**的区域，必须进行严格限制。

## 逐项风险分析

| 危险倾向               | 是否存在风险 | 风险描述                                                                 | 风险等级 |
|------------------------|--------------|--------------------------------------------------------------------------|----------|
| Knowledge Explanation  | 高           | 对资产进行“这个图解释了什么原理”“这个表展示了什么规律”的解释性描述     | 高       |
| Semantic Summarization | 高           | 对资产内容进行总结性归纳（如“该流程图总结了细胞呼吸的全过程”）           | 高       |
| Concept Discovery      | 中           | 从资产中自动识别“核心概念”，并尝试建立与知识点的关联                       | 中-高    |
| Teaching Interpretation| 高           | 判断资产的“教学价值”“适合哪个知识点”“难度等级”“是否为重点”             | 高       |

## Asset Recovery Layer 的正确边界

**可以做（属于 Document Recovery）**：
- 识别资产在页面中的位置与边界
- 恢复资产的结构信息（表格的行列结构、公式的符号结构、图表的组成元素）
- 提取资产的原始标题、编号、位置语义（“左上角图”“与上文公式对应”）
- 建立资产与周围正文、其他资产的引用关系
- 生成资产的基础描述性元数据（类型、尺寸、是否包含文字、是否为复合资产）
- 记录完整的 Provenance 链路（从原始页面到恢复结果的全过程）
- 标注资产的恢复质量（清晰度、是否被遮挡、OCR 置信度等）

**严禁做（进入 Knowledge Engineering）**：
- 对资产进行任何形式的“意义解释”或“知识归纳”
- 判断资产“说明了什么概念”或“对应哪个知识点”
- 进行教学化解读（“这个图适合帮助学生理解……”）
- 生成面向学习的总结、要点提炼或教学建议
- 主动与外部知识体系建立语义关联

**核心原则**：
Asset Recovery Layer 的输出必须是“资产被恢复后的客观事实 + 结构关系”，而非“资产被理解后的知识内容”。

# 问题4 审计：DIS 与下游职责链的边界合理性

## 审计结论

**当前提出的边界划分是合理的，且具备长期演进价值。**

### 职责链划分评估

| 维度                 | DIS（Recover → Normalize → Assemble → Trace） | 下游（TeachingChunker → KnowledgeStructurer → ...） | 评价 |
|----------------------|-----------------------------------------------|-----------------------------------------------------|------|
| 单一职责原则 (SRP)   | 高度符合                                      | 高度符合                                            | 优秀 |
| 长期演进能力         | 极强（恢复能力可独立演进）                    | 极强（知识策略可完全独立迭代）                      | 优秀 |
| 跨领域适用性         | 极强（软考、PMP、法律、医学、金融等均适用）   | 强（各领域可在此底座上发展专属知识工程）            | 优秀 |
| 认知边界清晰度       | 清晰（“恢复教材事实” vs “构建领域知识”）      | 清晰                                                | 优秀 |
| 变更影响隔离         | 优秀（DIS 变更不影响知识策略，反之亦然）      | 优秀                                                | 优秀 |

### 关键判断

1. **符合单一职责原则**：是的。DIS 只对“教材被恢复到什么程度”负责，下游只对“如何把恢复后的文档转化为可用知识”负责。

2. **符合长期演进原则**：是的。这种分离使得 DIS 可以持续提升恢复精度（OCR、Layout、跨页关系、复杂图表恢复等），而不会被任何特定知识工程范式（RAG、GraphRAG、Agent Memory 等）所绑架。

3. **适合作为多领域统一底座**：高度适合。无论未来是医学教材、法律教材还是金融教材，底层对“结构、资产、引用、溯源”的恢复需求是高度共性的。只有把这些共性收敛到 StructuredDocument 这个 Canonical Model 中，上层才能真正实现领域知识工程的差异化发展。

**唯一需要警惕的风险**：下游组件试图反向要求 DIS “多做一点知识相关的工作”（例如要求 DIS 预先切好 Chunk 或标注重点），这必须被坚决拒绝。

# 问题5（最高优先级）：DIS 的永久边界（Permanent Boundary）

## 审计结论

这是本次审计最核心的输出。

DIS 的永久边界必须以“**是否属于教材事实的恢复与规范化**”作为唯一判断标准。

## DIS 永久边界定义

### 必须永远属于 DIS（Permanent Inside）

无论技术如何发展、无论下游需求如何强烈，以下内容**必须永远留在 DIS 内部**：

1. **所有教材事实的恢复能力**
   - OCR（含公式、表格特殊识别）
   - Layout Analysis 与 Reading Order Recovery
   - 结构层级恢复（TOC、Section、Hierarchy）
   - 资产恢复（Figure、Table、Formula、Diagram）
   - 交叉引用恢复（Cross Reference Recovery）
   - 任何未来可能出现的新的“可从扫描教材中恢复的客观事实”类型

2. **规范化与组装能力**
   - 将恢复后的碎片组织为一致的、符合 MCDM 的 StructuredDocument
   - 冲突消解、重复消除、结构补全

3. **全链路溯源能力（Trace）**
   - 每一个对象的完整 Provenance 链路
   - 恢复过程的完整审计轨迹
   - 质量与置信度声明

4. **StructuredDocument 作为 Canonical Model 的定义权与完整性保障**
   - 决定 MCDM 包含什么、不包含什么
   - 保证 StructuredDocument 的自洽性与封闭性

### 无论诱惑多大都绝不能进入 DIS（Permanent Outside）

以下内容**无论任何理由，都必须永远排除在 DIS 之外**：

1. **任何形式的知识抽取与建模**
   - Concept / Knowledge Point Extraction
   - 知识关系建模（前置、包含、对比等）

2. **任何消费策略相关的工作**
   - Chunk Generation 及 Chunk Strategy
   - Embedding 相关决策
   - Retrieval 优化（稀疏、密集、混合、重排序等）

3. **任何解释性与教学性工作**
   - Semantic Interpretation / Knowledge Explanation
   - Teaching Interpretation（教学价值判断、学习目标映射）
   - 领域知识总结与提炼

4. **任何下游系统优化相关的内容**
   - 为特定 RAG 范式做的预处理
   - 为特定 Agent 框架做的适配
   - 为特定向量数据库做的格式优化

5. **任何“意义”层面的判断**
   - 重要性判断（“这是重点”）
   - 难度判断
   - 适用场景判断（“适合出题”“适合讲解”）

## 永久边界判断准则（供未来十年使用）

**判断一个新功能是否应该进入 DIS 的唯一标准**：

> 这个功能是否在回答“教材里客观存在什么、它在什么位置、它和其他部分是什么结构关系”这个问题？

- 如果回答“是”，则属于 DIS。
- 如果回答涉及“这个东西意味着什么、应该怎么用、适合怎么切、对应什么知识”，则**坚决不属于** DIS。

## 综合审计结论与建议修正项

### 架构审计结论

1. StructuredDocument 仍存在轻微的“中间产物思维”残留，需要进一步在定位描述中根除。
2. MCDM 的七类一级对象定义基本正确，但需明确其作为“下游唯一事实来源”的绝对地位。
3. Asset Recovery Layer 是当前最高风险区域，必须建立更严格的“禁止解释”红线。
4. DIS 与下游的职责链划分合理，具备成为多领域教材统一底座的潜力。
5. 永久边界的定义已基本清晰，核心在于严格执行“事实恢复” vs “知识工程”的切割。

### 建议修正项（优先级排序）

| 优先级 | 修正项 | 说明 |
|--------|--------|------|
| P0     | 在所有架构文档中彻底清除“StructuredDocument 是为知识工程准备的中间产物”相关表述 | 根除中间产物思维 |
| P0     | 明确声明 StructuredDocument 是 MCDM，所有下游必须以其为唯一事实来源 | 确立 Canonical 地位 |
| P1     | 为 Asset Recovery Layer 制定“禁止进入知识解释领域”的硬性约束清单 | 防止隐性越界 |
| P1     | 将“永久边界判断准则”写入架构决策记录，作为未来功能评审的强制检查项 | 十年长期约束机制 |

## 最终边界定义（DIS v1.1 永久冻结版）

**DIS 的永久使命**：

> 通过 Recover、Normalize、Assemble、Trace，将扫描版教材 PDF 恢复为一个自洽、完备、可精确追溯的 **StructuredDocument**（Minimum Canonical Document Model），作为所有下游知识工程工作的唯一客观事实来源。

**DIS 永不触碰的领域**：

> 任何涉及知识抽取、意义解释、消费策略优化、教学价值判断的工作，均永久排除在 DIS 边界之外。

**本次审计目标达成**：

DIS 的永久边界（Permanent Boundary）已在本轮审计中完成定义与冻结。
