# DIS v1.1 MCDM — StructuredDocument Schema 定义文档（Phase 1 冻结版）

**版本**：v1.0.0 Schema Freeze  
**阶段**：Phase 1 — Schema Freeze（最高优先级）  
**状态**：不可变（Immutable for v1.x）  
**批准依据**：DIS v1.1 架构最终审计（Permanent Boundary Audit）已通过

**核心原则**：
- 本文档定义的是 **Minimum Canonical Document Model (MCDM)**。
- StructuredDocument 是 DIS 的**唯一最终输出物**，也是所有下游系统的**唯一事实来源**。
- 所有设计严格遵守 Permanent Boundary：仅包含教材事实恢复结果，严禁任何知识语义内容。

---

## 1. 模型定位（不可修改）

**StructuredDocument** 是扫描版教材 PDF 经过 DIS 处理后，所能达到的**最小但完备的规范形态**（Canonical Representation）。

它必须满足以下铁律：

1. **自洽性**：无需依赖原始 PDF 即可完整重建教材的结构、资产、引用关系和来源。
2. **可追溯性**：每一个可引用对象都必须携带完整 Provenance。
3. **事实性**：仅记录“教材中客观存在什么、在哪里、有什么结构关系”，不记录任何“意义”、“重要性”或“如何使用”。
4. **封闭性**：下游系统（TeachingChunker、KnowledgeStructurer 等）不得需要回退原始 PDF 进行二次解析。

---

## 2. 一级对象（Canonical Objects）定义

StructuredDocument 由以下七个**一级对象**构成。每个对象都有明确的职责边界。

### 2.1 DocumentMeta

**职责**：记录教材的身份、来源和整体处理元数据。

**必须包含的核心信息**：
- 文档唯一标识（document_id）
- 原始文件信息（文件名、页数、格式、语言）
- 处理上下文（处理时间、DIS 版本、处理配置摘要）
- 整体文档级别 Provenance 记录

**禁止内容**：
- 任何教学相关元数据（学科、年级、难度、适用对象等）
- 任何知识体系归属信息

### 2.2 Structure（Chapter Tree）

**职责**：表达教材的完整逻辑层级结构。

**必须包含的核心信息**：
- 完整的章节树（Chapter → Section → Subsection → …）
- 每个节点的标题、层级编号、页码范围
- 目录恢复结果与实际正文结构的对应关系（是否从目录页恢复、是否推断）
- 节点之间的包含关系

**禁止内容**：
- 任何“知识模块”或“学习单元”的概念
- 节点的重要性、顺序建议等

### 2.3 ContentBlocks

**职责**：按阅读顺序组织的逻辑内容单元。

**必须包含的核心信息**：
- 按逻辑阅读顺序排列的内容块序列
- 内容块的基本分类（正文段落、标题、列表、引用块、公式行等）
- 内容块与页面的对应关系
- 跨页、跨章节的内容连续性标记

**禁止内容**：
- 任何 Chunk 相关的切分信息
- 内容块的语义摘要或主题标签

### 2.4 Assets

**职责**：所有被恢复的视觉与结构化资产的集合。

**资产类型**（v1 范围）：
- Figure（图片、插图、照片）
- Table（表格）
- Formula（数学/化学/物理公式）
- Diagram（架构图、流程图、思维导图等）

**每个资产必须包含的核心信息**：
- 唯一身份标识（asset_id）
- 资产类型
- 页面位置（page + bbox）
- 标题/编号/位置语义（“图 3-2”“左上角表格”等）
- 结构恢复结果（表格行列结构、公式符号序列、图像边界等）
- 与正文及其他资产的引用关系
- 基础恢复元数据（清晰度、是否复合资产等）
- 完整 Provenance

**禁止内容**：
- 资产的语义解释（“该图展示了光合作用的过程”）
- 教学价值判断
- 与知识点的关联

### 2.5 CrossReferences

**职责**：教材内部所有交叉引用关系的集合。

**必须包含的核心信息**：
- 引用源（文本位置或资产）
- 引用目标（图、表、公式、章节、页面等）
- 引用类型（“见”“如……所示”“公式”“表”等）
- 引用关系的可信度与恢复方式（显式文字、位置推断等）
- 双向索引（正向引用 + 反向被引用）

**禁止内容**：
- 引用关系的知识含义解释

### 2.6 Provenance

**职责**：全文档统一的可追溯性体系（可作为独立对象或嵌入其他对象）。

**必须支持的追溯能力**：
- 每一个可引用对象（结构节点、内容块、资产、引用）都必须关联 Provenance 记录
- Provenance 至少包含：
  - 来源页面与物理位置（page + bbox）
  - 恢复来源阶段（Ingestion / Structural Recovery / Asset Recovery）
  - 恢复所使用的模型/规则版本摘要
  - 置信度评分
  - 异常/降级记录（如 OCR 质量差、推断而非显式）

**设计要求**：
- Provenance 必须形成可机器解析的完整链路。
- 支持从最终 StructuredDocument 中的任意对象反向追溯到原始 PDF 区域。

### 2.7 IntegrityManifest

**职责**：文档整体恢复质量的结构化声明。

**必须包含的核心信息**：
- 整体恢复完成度（结构完整度、资产覆盖率、引用完整率）
- 各主要部分的恢复质量分布
- 明确标注的缺失或严重降级区域
- 处理过程中的关键决策记录（例如“目录页缺失，采用正文标题推断”）
- 全局置信度评估

**作用**：
为下游系统提供“这个 StructuredDocument 能信任到什么程度”的客观依据，而非主观评价。

## 3. 模型级强制约束（v1 冻结）

1. **零知识语义原则**  
   所有一级对象及其字段均不得包含任何 Concept、知识点、教学意义、重要性、难度等语义信息。

2. **全对象可追溯原则**  
   任何在 StructuredDocument 中可被独立引用的对象，都必须携带或关联 Provenance。

3. **事实优先原则**  
   当恢复完整性与表达美观/简洁发生冲突时，优先保证教材事实的完整性和可追溯性。

4. **封闭性原则**  
   StructuredDocument 必须是自描述的。下游系统不得需要额外访问原始 PDF 才能理解其内容组织和引用关系。

5. **不可变性原则（Schema 层面）**  
   v1.0.0 Schema 一旦冻结，在 1.x 版本中只允许 additive 变更（新增可选字段），禁止修改或删除已有字段的语义。

---

## 4. Schema 演进策略（v1 系列）

- **1.0.x**：仅修复性描述优化、字段补充说明，不改变模型语义。
- **1.1+**：允许在明确扩展区域增加新可选字段，但必须保持 MCDM 的最小核心不变。
- 任何可能影响下游依赖的变更，必须走正式的 Schema 变更评审流程。
